# Plan: Create CLI Commands for 4 Local Skills

## Summary
Create 4 new Kilo command files in `.kilo/command/` that wrap the existing local skills. Each command follows the same pattern as `git-backup.md` / `git-pages.md`: YAML frontmatter, usage info, and a workflow that loads the associated skill.

**Confirmation**: `publish-to-github-pages` skill is superseded by `/git-pages` (which deploys all slideshows at once with a landing page). No separate command needed.

**Not creating**: `/typst-author` (as requested).

## Files to Create

### 1. `.kilo/command/create-pdf-lesson-file.md`

- **Description**: "Converts a lesson plan JSON file to a formatted PDF using Typst CLI and Jinja2 templating."
- **Workflow**:
  1. Accepts JSON file path as argument (the output JSON from `write-lesson-plan`)
  2. Validates: JSON file exists, meets schema
  3. Loads the `create-pdf-lesson-file` skill via `skill` tool
  4. Runs: `python scripts/json_to_pdf.py <json_path>`
  5. Reports output path: `PDF/{subfolder}/{mmddyy}-{topic}-lesson-plan.pdf`

### 2. `.kilo/command/lesson-plan-to-reveal.md`

- **Description**: "Converts a lesson plan JSON into a reveal.js presentation using raw HTML section elements. Markdown pipeline is permanently abandoned — uses the base template from `templates/base-slides-template.html`."
- **Workflow**:
  1. Accepts JSON file path as argument
  2. Validates: JSON exists, output dir exists
  3. Loads the `lesson-plan-to-reveal` skill via `skill` tool
  4. Copies base template: `cp "templates/base-slides-template.html" "output/{subfolder}/slides/index.html"`
  5. Copies logo to `assets/`
  6. Builds slides as raw HTML `<section>` elements
  7. Reports output path

### 3. `.kilo/command/slideshow-to-pptx.md`

- **Description**: "Converts a reveal.js HTML slideshow to a full-visual-fidelity PowerPoint .pptx file using Decktape (Chrome headless → PDF) and PyMuPDF (PDF pages → PPTX slides)."
- **Workflow**:
  1. Accepts subfolder name as argument
  2. Verifies dependencies: `node --version`, `decktape --version`, Python packages
  3. Loads the `slideshow-to-pptx` skill via `skill` tool
  4. Creates `pptx/` output directory
  5. Runs Decktape to render slides as PDF
  6. Runs `python scripts/pdf_to_pptx.py` to build PPTX
  7. Reports output path

### 4. `.kilo/command/write-lesson-plan.md`

- **Description**: "Interactively gathers teacher requirements and generates a lesson plan in JSON format. Asks one question per turn using chat or the question tool."
- **Workflow**:
  1. Loads the `write-lesson-plan` skill via `skill` tool — this skill owns the entire interactive workflow
  2. Follows the skill's 13-step interactive workflow (greeting, shape selection, teacher name, duration, CEFR, etc.)
  3. Outputs JSON to `output/{input_subfolder}/{mmddyy}-{topic}-lesson-plan.json`

## Pattern for all commands

```yaml
---
description: "<concise description matching the skill's description>"
---
# Command: <Name>

## Usage
`/<command-name> [<args>]`

## What it does
<1-3 line summary>

## Prerequisites
<if any>

## Workflow
### Step 1: Load skill
```
skill create-pdf-lesson-file
```
### Step 2: ...
```

## Edge Cases (per command)
- **create-pdf-lesson-file**: JSON missing fields → validation error; answer_key path not found → warn and fallback to "none"; Typst compile fail → show stdout/stderr
- **lesson-plan-to-reveal**: Missing JSON → abort; missing template → error; output dir missing → create it
- **slideshow-to-pptx**: Missing slides → abort; decktape not installed → install instruction; node not found → error
- **write-lesson-plan**: User cancels mid-flow → no partial file written; shape JSON missing → error; invalid date → use today
