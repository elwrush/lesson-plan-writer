# Plan: Skip mkslides + Stabilize Slides — Direct reveal.js + Partial Regeneration

## Problem

1. **Pipeline complexity** — mkslides adds an unnecessary build step (Jinja → HTML), copies 100+ unused highlight.js theme files (~30MB), and requires npm dependency
2. **Speaker notes exposed** — `Notes:` blocks appear as visible text on slides (mkslides doesn't configure `data-separator-notes`)
3. **Image instability** — Pixabay returns different images each run; cache gets polluted with unrelated images
4. **No partial updates** — any change requires full markdown regeneration, overwriting the entire file

## Solution Architecture

**Three-part fix:**

### Part A: Skip mkslides, use reveal.js directly (inline markdown)
- Create static `templates/slides-template.html` with CDN reveal.js + plugins
- `<section data-markdown data-separator-notes="^[Nn]otes?:">` with `<textarea data-template>{{ MARKDOWN }}</textarea>`
- `json_to_markdown.py` injects generated markdown into template → `output/subfolder/slides/index.html`
- Single self-contained file, opens from `file://` (no server), refresh to see changes
- Reveal.js markdown plugin natively parses `Notes:` → `<aside class="notes">` (students never see procedure text)

### Part B: Deterministic Generation (prevent unintended changes)
Make re-running produce identical output by stabilizing all variable elements.

### Part C: Partial Regeneration (update only targeted sections)
Add `--section` flag to regenerate specific slides only, merging into the existing markdown + regenerating `index.html`.

---

## New Pipeline

```
JSON → json_to_markdown.py → output/subfolder/slides/
                               ├── slides.md        (intermediate, human-readable)
                               ├── index.html       (self-contained reveal.js)
                               └── assets/          (logo, Pixabay images)
```

**Open**: Double-click `index.html` in any browser, or push to GitHub Pages.
**Refresh**: Re-run `json_to_markdown.py`, refresh browser.

GitHub Pages deployment: push `output/subfolder/slides/` contents to `gh-pages` branch. Single HTML file + assets. No build pipeline on server.

def get_cache_key(query):

### Step 6: Add `write_index_html()` to json_to_markdown.py

**File:** `scripts/json_to_markdown.py`

```python
SLIDES_TEMPLATE = PROJECT_ROOT / "templates" / "slides-template.html"

def write_index_html(markdown_content, output_dir):
    """Write a self-contained index.html with markdown inlined."""
    template = SLIDES_TEMPLATE.read_text(encoding="utf-8")
    html = template.replace("{{ MARKDOWN }}", markdown_content)
    index_path = output_dir / "index.html"
    index_path.write_text(html, encoding="utf-8")
    print(f"index.html written: {index_path}")
```

Called from `convert_json_to_markdown()` after generating/replacing markdown:
- Full generation: generate markdown → write `slides.md` → write `index.html`
- Partial generation (`--merge`): merge sections → write `slides.md` → read markdown → write `index.html`

## Expected Behavior

1. **Single command** — `json_to_markdown.py` produces both `slides.md` and `index.html`
2. **No server needed** — open `index.html` directly in Firefox/Chrome, refresh to see changes

4. **Partial updates** — `--section title --merge` updates only the title slide, everything else preserved

6. **No mkslides** — zero build step, zero npm dependencies in output
7. **GitHub Pages ready** — push `slides/` contents to `gh-pages`, live at `https://{user}.github.io/{repo}/`
