# Plan: Deterministic Slide Editing

## Problem

When the user asks to edit a slide at a reveal.js URL (e.g., `index.html#/7`), the agent must manually translate the hash index to a slide section. This is error-prone — the agent can confuse `transition-2` with `prereading`, or edit the wrong `<!-- slide: N -->` block.

## Solution: `scripts/locate_slide.py`

A small standalone Python script that maps a reveal.js hash index to the corresponding slide section in the markdown file.

### Usage

```bash
# From a full URL
python scripts/locate_slide.py "file:///C:/PROJECTS/LESSON PLAN WRITER 3/output/substitute-lesson-1-M3/slides/index.html#/7"

# From a hash alone (auto-discovers markdown from slides/ dir)
python scripts/locate_slide.py 7 --slides-dir output/substitute-lesson-1-M3/slides/

# From a markdown file path + hash
python scripts/locate_slide.py 7 --md output/substitute-lesson-1-M3/slides/050726-what-connects-us-slides.md
```

### Output (JSON to stdout)

```json
{
  "slide_index": 7,
  "section": "transition-2",
  "heading": "What's the idea?",
  "slide_comment_line": "<!-- slide: 7 -->",
  "lines": {
    "start": 85,
    "end": 100
  },
  "content": "<!-- slide: 7 -->\n<!-- slide-section: transition-2 -->\n..."
}
```

### How it works

1. Parse input — extract hash index from URL, or accept index directly
2. Locate the markdown file — either from `--md` arg, from `--slides-dir` (find `*-slides.md`), or from the URL path (replace `index.html` with the `*-slides.md` file in same dir)
3. Parse `<!-- slide: N -->` comments to build an index → line-number map
4. Find the section containing the requested slide index
5. Extract the section name from `<!-- slide-section: NAME -->`
6. Output JSON to stdout so the agent can parse it deterministically

### Why JSON output

The agent can pipe output to parse and get exact line numbers, section names, and content without regex guesswork. This eliminates the "edit the wrong slide" error class entirely.
