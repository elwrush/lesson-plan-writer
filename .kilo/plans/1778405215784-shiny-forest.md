# Plan: Fix Image Regeneration Problem

## Root Cause Summary

`json_to_markdown.py` regenerates EVERYTHING from scratch on every run — including all image references. It has no `--text-only` mode and no image-preservation logic. Running it against production output **always overwrites image references** because:

1. The script searches Pixabay (or `.image-cache/`) for background images
2. It picks the first matching file without knowing which specific image the user wants
3. It writes those references into the generated markdown, replacing whatever was there before

The voice changes I made were correct. The mistake was regenerating production slides to test them, which triggered the broken image pipeline.

## The Fix: Text-Only Regeneration Mode

Add a `--no-images` flag to `json_to_markdown.py`. When this flag is set:
- The script reads the EXISTING markdown file to extract current image references
- It preserves those exact image references in the regenerated output
- It never calls `search_and_get_image()` or `find_existing_asset()`
- It only regenerates text content (voice, vocabulary, objectives, etc.)

### Implementation

**1. Add `--no-images` CLI argument**
```python
parser.add_argument("--no-images", action="store_true",
                    help="Preserve existing image references — text-only regeneration")
```

**2. Add `extract_image_references(existing_md)` function**
- Parses the existing markdown file
- Extracts all `data-background-image` values
- Returns a dict mapping `slide-section` → image path

**3. Modify slide generators to accept optional `image_path` override**
- If `image_path` is provided (from extracted references), use it directly
- If not provided AND `--no-images` is set, use gradient fallback
- If not provided AND `--no-images` is NOT set, use existing search logic

**4. Update `convert_json_to_markdown()` flow**
```
if --no-images:
    existing_md = read current markdown file
    image_map = extract_image_references(existing_md)
    pass image_map to generate_markdown()
else:
    normal flow (search Pixabay, etc.)
```
