# Plan: Create `/git-backup` and `/git-pages` Kilo Commands

## Design Decisions

- **Both commands are "full"** — when you invoke them, they actually do the work (stage, commit, push). This is safe because you're explicitly requesting the action.
- **Commands go in `.kilo/command/`** — tracked by git, no restart needed, team members get them on clone.
- **Existing `publish-to-github-pages` skill** at `.kilocode/skills/publish-to-github-pages/SKILL.md` is retained as reference but the command becomes the primary interface.
- **Both commands run `/lint` first** — auto-lint before any commit, ensuring no un-linted code gets pushed.

---

## Command 1: `/git-backup`

**Purpose:** Stage all changes, show a summary, prompt for a commit message, commit to main, and push to origin.

### Workflow
1. Run `/lint` (ruff check + format)
2. `git status` — show overview of changed files
3. `git add -A` — stage everything
4. `git diff --cached --stat` — show summary of staged changes
5. Auto-generate commit message from changed filenames:
   - 1 file: `"Update {file}"` (e.g. `"Update scripts/json_to_pdf.py"`)
   - 2–4 files: `"Update {file1}, {file2}"`
   - 5+ files: `"Update multiple: {file1} and N more files"`
   - If only untracked additions: `"Add {file1}, {file2}"`
   - If only deletions: `"Remove {file1}, {file2}"`
6. Print the auto-generated message and ask: "Commit with this message? (y/n)"
   - If 'y': proceed
   - If 'n': prompt for custom message instead
7. `git commit -m "{message}"`
8. `git push origin main`
9. Print: "Backup complete. main is {N} ahead of origin/main."

### Edge cases
- If nothing to commit: stop with message "Nothing to commit — working tree clean."
- On push failure: report error, don't lose the commit

### Design intent
The `/lint` prefix ensures code is clean before backup. The stat summary gives you a sanity check. The message prompt enforces meaningful commits.

## Command 2: `/git-pages <subfolder>`

**Purpose:** Publish a slideshow to GitHub Pages by adding/updating a subfolder on the gh-pages branch.

**Parameter:** `<subfolder>` — the topic subfolder name (e.g., `welcome-to-term-1`)

### Prerequisites
- gh CLI authenticated: `gh auth status`
- Slides exist at `output/{subfolder}/slides/index.html`
- Remote origin is a GitHub repo

### Workflow
1. Validate input: `<subfolder>` is provided and slides exist
2. Run `/lint`
3. Detect remote owner/repo from `git remote get-url origin`
4. Save current branch (`main`)
5. Copy slides to temp directory: `$env:TEMP\gh-pages-deploy-{subfolder}`
6. Fetch gh-pages branch: `git fetch origin gh-pages` (in case it's new on remote)
7. Switch to gh-pages: `git checkout gh-pages`
8. Create/update subfolder: copy temp contents into `{subfolder}/`
9. **Update root `index.html`** — append or create a landing page that lists all presentation subfolders (read existing root index.html if present, parse subfolder list from `git ls-tree --name-only`, regenerate landing page)
10. `git add -A`
11. `git commit -m "Add slides: {subfolder} ({date})"`
12. `git push origin gh-pages`
13. Return to `main`: `git checkout main`
14. Print: `Slides published at: https://{owner}.github.io/{repo}/{subfolder}/`

### Landing page (root index.html — card grid)
The root of `gh-pages` shows a styled card grid, NOT a slideshow. Generated by the command:
- Read existing gh-pages tree to find all subdirectories containing `index.html`
- Generate a card grid page with each presentation as a card: title, date, link
- Styled: centered layout, card background, hover effect
- Title: "Lesson Plan Slides"

### First-ever deploy (orphan branch)
The command handles this automatically:
- If `origin/gh-pages` doesn't exist: create orphan branch, copy slides to root (single-site), commit, force-push
- If `origin/gh-pages` exists: use subfolder workflow above
- No manual first setup required

### gh-pages branch doesn't exist locally
- Fetch from origin first: `git fetch origin gh-pages`
- If still missing: create orphan branch (first deploy)

### Edge cases
- gh-pages branch doesn't exist locally: `git checkout -b gh-pages origin/gh-pages` or create orphan
- Slides dir missing: error with message "No slides found at output/{subfolder}/slides/"
- gh not authenticated: error with instruction to run `gh auth login`
- Push fails: report error, stay on gh-pages branch so user can debug manually
- Temp copy fails: abort before switching branches
