# Remove Automatic Pixabay from json_to_markdown.py

## Problem
The slide generator automatically downloads Pixabay images when generating slides. This causes:
- Non-deterministic image selection — same JSON → different images each run
- Fragile matching in `find_existing_asset()` picks wrong images when multiple candidates exist
- Side-effect pollution — `search_and_get_image()` + `copy_to_assets()` dump new files into `assets/` even when old ones should be preserved
- API dependency that silently fails when `PIXABAY_API_KEY` is unset

## Solution
**Remove all automatic Pixabay search/download logic from `json_to_markdown.py`**. The script should only:
1. Use gradient backgrounds by default (no image = gradient)
2. Check `assets/` for manually placed images via `find_existing_asset()`
3. Preserve existing image references on regeneration via `load_image_references()`/`restore_image_references()`

Images become a post-generation step: user/agent runs `pixabay_download.py` manually, places files in `assets/`, updates markdown references.

### 2. Update `generate_title_slide()` (line 430)

Current flow:
1. Checks `title_image_path` arg
2. Falls to `find_existing_asset(slides_dir)` — picks ANY non-logo image (wrong one often)
3. Falls to `search_and_get_image()` — downloads from Pixabay

New flow:
1. If `title_image_path` arg provided → use it
2. Else if `slides_dir` → call `find_existing_asset(slides_dir, "pixabay_486")` or specific prefix  
3. Else → gradient background

Better approach: Don't auto-search at all. If no image path is provided and no existing asset matches, use gradient. Remove lines 442-450 (the `search_and_get_image` call).

```python
def generate_title_slide(data, title_image_path=None, title_attribution=None, slides_dir=None, logo_path=None):
    ...
    existing = None
    if title_image_path:
        image_path = Path(title_image_path).resolve()
    elif slides_dir:
        existing = find_existing_asset(slides_dir, "pixabay_486")
        if existing:
            image_path = (Path(slides_dir) / existing).resolve()
        else:
            image_path = None  # ← gradient fallback, no Pixabay search
    else:
        image_path = None
    ...
```

Actually — simpler and more correct: remove ALL automatic image fetching. The user's workflow is post-generation placement. So `generate_title_slide()` should just check `find_existing_asset()` and fall back to gradient. No `search_and_get_image()` call at all.
