# Plan: Integrate reveal.js Audio Slideshow Plugin

## Goal

Add the `audio-slideshow` plugin from [rajgoel/reveal.js-plugins](https://github.com/rajgoel/reveal.js-plugins/tree/master/audio-slideshow) to enable audio playback during presentations. This allows the teacher to play embedded audio tracks (e.g., listening comprehension recordings, pronunciation, task instructions) directly within the reveal.js slideshow — no external media player needed.

## Plugin Summary

**CDN URL (playback only, not recording):**
```
https://cdn.jsdelivr.net/npm/reveal.js-plugins@latest/audio-slideshow/plugin.js
```

**Key features:**
- Play audio via `data-audio-src` attribute on `<section>` elements or fragments
- Optional auto-advance to next slide after audio finishes (configured via `data-audio-advance`)
- Audio player controls overlay (opacity 0.05 when unfocused, 1.0 on hover)
- Keyboard shortcut: `A` to toggle audio pause/play
- Text-to-speech fallback (not needed for lesson plans — teacher uses pre-recorded audio)
- Audio files stored in `audio/` subdirectory

**Recording addons** (`recorder.js`, `RecordRTC.js`) are NOT needed — the teacher will provide pre-recorded audio files or record externally.

## Changes

### 1. Base Template (`templates/base-slides-template.html`)

Add before `</head>`:
```html
<!-- Audio slideshow plugin -->
<script src="https://cdn.jsdelivr.net/npm/reveal.js-plugins@latest/audio-slideshow/plugin.js"></script>
```

Add `RevealAudioSlideshow` to plugins array and configure `audio` options:
```js
Reveal.initialize({
    // ...existing config...
    audio: {
        prefix: 'audio/',
        suffix: '.mp3',
        advance: -1,   // don't auto-advance (teacher controls pacing)
        autoplay: false,
        defaultAudios: false,  // only explicit data-audio-src
        defaultDuration: 0,    // no silent filler
        playerOpacity: 0.05,
        playerStyle: 'position: fixed; bottom: 4px; left: 25%; width: 50%; height:75px; z-index: 33;',
        startAtFragment: false,
    },
    plugins: [
        RevealHighlight,
        RevealMath.KaTeX,
        RevealNotes,
        RevealSearch,
        RevealZoom,
        RevealAudioSlideshow,   // ← add
    ],
});
```

## Usage Patterns

```html
<!-- Slide with audio that teacher triggers -->
<section data-audio-src="audio/listening-track.mp3">
    <h2>Listen to the Podcast</h2>
    <p>Track 1.02 — YouTubers and Teenagers</p>
</section>

<!-- Fragment with audio -->
<p class="fragment" data-audio-src="audio/pronunciation.mp3">
    <span class="vocab-word">persuade</span> /pəˈsweɪd/
</p>
```

## Edge Cases & Notes

- **No audio files yet**: Plugin is inert until `data-audio-src` attributes are added. Adding the plugin doesn't affect existing slides.
- **Browser support**: Playback works on recent desktop Firefox and Chrome. HTTPS recommended for production.
- **Audio format**: MP3 widely supported. Plugin also handles `.ogg` and `.webm`.
- **gh-pages deployment**: Audio files are static assets — `/git-pages` deploys them automatically with the rest of `slides/`.
- **Timer plugin compatibility**: No conflict — timer uses its own `<audio>` element for the chime. Audio-slideshow creates per-slide `<audio>` elements.
- **Player visibility**: Controls appear at bottom of viewport with 5% opacity (barely visible). Hover reveals full controls.
