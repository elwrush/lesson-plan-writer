# Plan: Fix Git Infrastructure Problems

## Problems Identified

### 1. `.git/index.lock` Contention (constant)

## Sources of `.git/index.lock` Contention in This Environment

**You use:** Kilo CLI (not VS Code), PowerShell, gh CLI, git, pre-commit hooks (ruff + pytest).

Likely sources of background git activity:
- **Kilo CLI itself** runs `git status`, `git log`, `git diff`, `git checkout`, `git add`, `git commit`, `git push` during normal operations. If Kilo's turn overlaps with a background process, lock contention occurs.
- **PowerShell prompt** — `posh-git` or `oh-my-posh` (if installed) runs `git status` on every prompt return, creating a lock window.
- **`gh` CLI** — `gh auth status` or other gh commands access `.git/` indirectly.
- **Windows Defender / real-time scanning** — holding `.git/index` open for scan while a second process tries to write to it.
- **Pre-commit hooks (ruff + pytest)** — install via `.pre-commit-config.yaml`. When running, they touch the git index. Even after `uninstall`, the config file remains and could be re-triggered by any `git commit` call.

**Key constraint:** no auto-commit, no auto-push. All commits and pushes are manual decisions.

**Fix:**
- Check for posh-git: `Get-Module -ListAvailable posh-git`. If present, it runs `git status` on every prompt — disable it during sessions or uninstall.
- Keep pre-commit **permanently uninstalled** unless you explicitly ask for it: `python -m pre_commit uninstall`
- Consider adding `.git/index` to Windows Defender exclusion list
- For Kilo's own git commands: add a `Start-Sleep -Seconds 1` before any `git checkout` or `git add` to let background locks clear

### 2. Skills Folder Disappears on Branch Switch

**Root cause:** `.kilocode/skills/` is tracked in `main` but not in `gh-pages`. `git checkout gh-pages` removes it from disk.

**Fix options (choose one):**

**Option A (Recommended): Stop tracking `.kilocode/` in git entirely.**
- `.kilocode/` is Kilo tooling configuration — not part of the project source code
- Add `.kilocode/` to `.gitignore`
- Remove from tracking: `git rm --cached -r .kilocode/`
- This prevents the skills folder from ever being affected by branch switches
- Skills can live in user-level config (`~/.config/kilo/skills/` or `~/.kilo/skills/`) for global availability

**Option B: Commit `.kilocode/skills/` to gh-pages branch too.**
- Simpler but bloats the gh-pages branch with tooling files that don't belong there

**Option C: Use `git worktree` for gh-pages.**
- Check out gh-pages into a separate directory (e.g., `../gh-pages-worktree/`)
- `main` stays checked out in the main project dir, skills folder never gets removed
- Worktree has its own `.git/index.lock`, so no lock contention with main either
- More setup but cleanest long-term solution

### 3. Workflow Fragmentation

**Root cause:** Switching between `main` and `gh-pages` disrupts the working tree (different files, different tracked paths).

**Fix:**
- Minimize branch switching during a session
- If using `git worktree` for gh-pages, you never need to leave `main`
- If not: commit/stash all work on `main` before switching to gh-pages; switch back immediately after push

## Recommended Action Plan

1. **Return to `main` first** — `git checkout main` (restores the skills folder from git's object store)

2. **Stop tracking `.kilocode/` in git** (manual steps):
   - Add `.kilocode/` to `.gitignore`
   - `git rm --cached -r .kilocode/`
   - Manual commit: `git commit -m "Stop tracking .kilocode/ — tooling config, not source"`

3. **If gh-pages work is needed**, use `git worktree` to avoid branch switching:
   - `git worktree add ../lpw-gh-pages gh-pages`
   - All gh-pages work happens in `../lpw-gh-pages/`
   - `main` stays checked out — skills folder never disappears
   - Each worktree has its own `.git/index.lock`, eliminating cross-branch lock contention

4. **Disable posh-git** (if present):
   - Check: `Get-Module -ListAvailable posh-git`
   - Temporary disable: `Remove-Module posh-git -ErrorAction SilentlyContinue`
   - Permanent: uninstall with `Uninstall-Module posh-git`

5. **Keep pre-commit uninstalled** — run `python -m pre_commit uninstall` after any session that requires it
