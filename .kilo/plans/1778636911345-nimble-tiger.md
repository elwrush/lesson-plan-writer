# Plan: Essay Paper — Grid-Based Ruled Lines

## Approach

Use a `grid` with fixed row heights (`repeat(n, 24pt)`) and `table.hlines` for ruled lines. This is idiomatic Typst — no tiling, no page fill hacks, no manual v()/line() loops.

### How it works

1. Header + table render normally as page content
2. `block(height: 1fr)` + `layout(size => ...)` measures remaining vertical space on page 1
3. `calc.floor(remaining / 24pt)` gives the number of ruled lines
4. `grid(rows: repeat(n, 24pt), ..table.hlines(...))` fills the rest with ruled lines
5. `pagebreak()` → page 2 uses same grid pattern but fills entire content area

### Why grid, not tiling

- Grid rows have **exact** heights — no spacing drift
- `table.hlines` draws lines **between** rows — no overlap with content above
- Line count is **calculated**, not hardcoded — adapts to any margin/content size
- No background masking needed — ruled lines are foreground content, not a page fill

## File changes

**`typst/code/May-13-2026-essay-paper/index.typ`** — complete rewrite:

```typst
#import "../../../templates/mathayom-header.typ": mathayom-header

#set text(font: "Roboto", size: 11pt)
#set page(
  paper: "a4",
  margin: (x: 0.75in, top: 0.75in, bottom: 1in),
)

#let ls = 24pt

#let ruled-lines(n) = {
  grid(
    columns: 1,
    rows: repeat(n, ls),
    stroke: none,
    inset: 0pt,
    ..table.hlines(
      start: 0,
      end: n,
      stroke: 0.4pt + black,
    ),
  )
}

#mathayom-header()

#table(
  columns: (auto, 6cm),
  stroke: 0.5pt + black,
  inset: 8pt,
  align: left + horizon,
  [CLASS], [],
  [ID], [],
)

#v(12pt)

// Page 1: fill remaining space
#block(height: 1fr, layout(size => {
  ruled-lines(calc.floor(size.height / ls))
}))

#pagebreak()

// Page 2: fill entire content area
#layout(size => {
  ruled-lines(calc.floor(size.height / ls))
})
```

### Line-by-line explanation

| Line | Purpose |
|---|---|
| `#let ls = 24pt` | Line spacing constant — 24pt ≈ 8.5mm |
| `#let ruled-lines(n)` | Reusable function: grid with n rows of `ls` height, hlines between each |
| `grid(rows: repeat(n, ls))` | Creates n rows, each exactly 24pt tall |
| `table.hlines(start: 0, end: n)` | Draws horizontal lines at every row boundary |
| `stroke: 0.4pt + black` | Thin black ruled lines |
| `block(height: 1fr, layout(size => ...))` | `1fr` expands to fill remaining page; `layout` reports the size |
| `calc.floor(size.height / ls)` | How many lines fit in the remaining space |
| Page 2 `layout(size => ...)` | No `1fr` wrapper needed — layout at page root gives full content area |

### Why page 2 doesn't need `block(height: 1fr)`

From the `layout` docs: *"If it is placed directly into the page, it receives the page's dimensions minus its margins."* So `layout` at the root of page 2 gives the full content height directly.
