# Plan: Fix Slide Text Highlight Overflow in Fullscreen (F11)

## Root Cause Analysis

**Problem**: On Windows F11 fullscreen, text highlighting (`rgba(0,0,0,0.5)`) on slide elements spills beyond the visible slide area.

**Three contributing factors**:

1. **`overflow: visible` on slides container** — `css/reveal.scss` sets `.reveal .slides { overflow: visible; }`. When reveal.js rescales slides on fullscreen (via `transform: translate(-50%, -50%) scale(N)`), any element that fills 100% of its parent can visually extend beyond the slide bounds because nothing clips it.

2. **`display: block` on h2/h3** causes the `rgba` background to span the full section width. Combined with `padding: 0 60px` on `section[data-markdown]`, the background fills the content box. At certain aspect ratios, this extends beyond the visible clipped area reveal.js would normally need.

3. **`display: inline-block` on p/li** creates single-rectangle background boxes that don't properly wrap with multi-line text. When the slide scale changes, these boxes can overflow.

## Fix: `templates/slides-template.html`

Add `overflow: hidden` to each slide section to clip content at the section bounds — the standard approach used in slide frameworks to prevent content bleed.

### Change 1 — Add `overflow: hidden` to all slide sections

In the `<style>` block, add:

```css
.reveal .slides section {
    overflow: hidden;
}
```

This goes after the existing `section[data-markdown]` rule. It clips any child element whose background/padding extends beyond the section's computed bounds — exactly what happens when reveal.js rescales on F11 fullscreen.

### Change 2 — Fix inline-block p/li overflow with `box-decoration-break`

Replace the `display: inline-block` on `<p>` and `<li>` with `display: inline` + `box-decoration-break: clone` to get per-line background boxes that properly contain multi-line text:

```css
.reveal .slides section p,
.reveal .slides section li {
    display: inline;
    -webkit-box-decoration-break: clone;
    box-decoration-break: clone;
    /* ... rest unchanged ... */
}
```

This ensures that when text wraps, each line gets its own background box clipped to the line's width, preventing the single-rectangle inline-block overflow.

### Change 3 — Limit h2/h3 width for background containment

Add `box-sizing: border-box` and `width: 100%` (already implied by block display) and ensure padding doesn't cause overflow:

The h2/h3 rules (`display: block; padding: 0.2em 0.4em;`) already work correctly with `overflow: hidden` on the section. No additional change needed — Change 1 alone fixes the h2/h3 spill.

## Scope

Only `templates/slides-template.html` needs editing. Subsequent regenerations via `json_to_markdown.py` will pick up the fix automatically. No changes to `json_to_markdown.py` or `docs/slide-design-reference.md`.
