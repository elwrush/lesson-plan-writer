# Plan: Eliminate Jinja2 from the Typst Pipeline

## Summary

Replace the Jinja2 rendering layer (Python → Jinja2 → `.typ`) with Typst's native `json()` data loading and scripting. The pipeline collapses from 3 steps to 2:

```
Before: JSON → [Python(Jinja2) → .typ] → typst compile → PDF
After:  JSON → [typst compile with json()] → PDF
```

## eval() Investigation

**Finding**: `eval()` IS a documented, built-in Typst function. Verified against the packed Typst source (`crates/typst-library/src/foundations/mod.rs`):
```
global.define_func::<eval>();
#[func(title = "Evaluate")]
pub fn eval(source: str, mode: str, scope: dictionary) -> any
```
It has a full test suite (`tests/suite/foundations/eval.typ`) and a dedicated engine routine (`eval_string`). The docs note: "This function should only be used as a last resort."

**Limitation discovered**: `eval(content, mode: "markup")` fails when the string contains unescaped `#` or `$` characters (Typst's hard-coded code/math delimiters). This is a parser-state conflict — `eval()` re-parses raw strings through the full syntax pipeline, inheriting all delimiter sensitivity. Since answer key content (after md→typ conversion) contains `==`, `-`, `*bold*`, and sometimes `$`, this made `eval()` unreliable for answer key embedding. (Idiomatic workaround: wrap delimiters in `\#`, `\$`, or `raw()`/`text()` — but f-strings avoid the problem entirely.)

**Decision**: Use Python f-string generation instead of `eval()`. This eliminates Jinja2 without depending on `eval()`. The answer key content is embedded directly into the generated `.typ` file as plain markup — no runtime evaluation needed.

## Architecture Decision

**Original plan**: Native Typst template using `json()` + `eval(read())` for answer keys.  
**Problem found**: `eval()` has parsing limitations with `#` and `$` characters in markup mode, making it unreliable for answer key content.  
**Solution**: Python f-string generation. The `.typ` file is built directly in Python, embedding all data inline. This eliminates Jinja2 without depending on `eval()` at runtime.

Pipeline:
```
Before: JSON → Python(Jinja2) → .typ → typst compile → PDF
After:  JSON → Python(f-strings) → .typ → typst compile → PDF
```
